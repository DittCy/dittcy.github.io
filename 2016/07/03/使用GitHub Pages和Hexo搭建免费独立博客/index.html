  • Hexo

前言

这是一篇使用GitHub Pages和Hexo搭建免费独立博客的总结。
作为一个码农，白是白了点，但深知技术博客的重要性，几经周折了解并搭建起这个博客平台，希望能够以此为起点，开始我的技术博客之路。
这里主要介绍搭建过程及遇到的一些坑，网上的信息比较混乱，有些也不完整，每个人可能遇到的错误都不尽相同，这里也只能列出我所遇到的，具体问题还得具体看，当然，无外乎百度能都得到的一众问题，难度不大。

一、准备环境

  • Git
  • GitHub
  • GitHub Pages
  • Nodejs
  • Hexo
  • Markdown
  • Dropbox

Git及Nodejs 下载后一路回车安装即可。关于工具的概念性问题及用法请移步百度。

这里只简单说下GitHub Pages：

在自己的GitHub账号下创建一个新的仓库，命名为username.github.io（此即为你的博客域名，username自定义)。
GitHub Pages有几种类型，这里了解，而我所使用的是User Pages。简单来说，User Pages与Project Pages的区别是：

  • User Pages是用来展示用户的，而Project Pages是用来展示项目的。
  • 用于存放User Pages的仓库必须使用username.github.io的命名规则，而Project Pages则没有特殊的要求。
  • User Pages将使用仓库的master分支，而Project Pages将使用gh-pages分支。
  • User Pages 通过http(s)://username.github.io进行访问，而Projects Pages通过http(s)://username.github.io/projectname进行访问。

六、写作与维护

1. Hexo程序文件备份

由于GitHub上存放的只是供访问的页面文件，真正的Hexo程序文件及文章原始md文件都在本地，为了更有效的容灾，同时也便于多台设备间的协作，需要进行备份。网上很多人采用Git增加一个分支来管理Hexo程序文件，试了感觉实在是不好用，总之我放弃了……我选择了Dropbox进行文件同步协作，本身就一直在用Dropbox，所以没有额外的开销，缺点也很明显：需要翻墙……

七、绑定域名

GitHub提供自定义域名的服务

1. DNSPod配置域名解析

  • 添加两条A记录，分别指向192.30.252.153和192.30.252.154，记录值为空
  • 添加一条CNAME，指向username.github.io，记录值为www

2. 配置GitHub

  • 在你的仓库根目录下新建文件，命名为CNAME，不要加后缀名，文件内容为你的域名，例如wwww.dittcy.cc就可以了。
  • 需要注意的是，每次部署后CNAME文件就会被删除，这是因为Hexo部署为整个仓库全部替换文件。解决办法是，将CNAME文件添加到本地Hexo目录的public文件夹下，部署时就会连同CNAME文件一起部署到Git仓库了。
  • 也可以用此法为你的仓库添加README.md文件。
